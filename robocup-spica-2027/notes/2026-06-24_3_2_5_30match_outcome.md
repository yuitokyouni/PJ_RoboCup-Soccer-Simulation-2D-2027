# 2026-06-24 — 3-2-5 vs default helios, 30-match outcome

## TL;DR

First RESEARCH_GRADE tactical-contrast result in the repo.
**HELIOS_L (default 4-2-3-1) vs HELIOS_3_2_5 (custom normal-formation
3-2-5 wingback-fluid) → 30 matches, no statistically significant
difference.** The aggregator and `make compare` both agree.

This matches subjective observation: only `normal-formation.conf`
was replaced; `defense-formation.conf` and `offense-formation.conf`
are still default helios, so the 3-2-5 shape only appears during the
"normal" play phase. The remaining phases revert the team to a
4-2-3-1, blunting whatever advantage the new normal-phase shape
might confer.

## Why the variant did not improve (and probably should not have, yet)

`experiments/helios_3_2_5_formations/` overrides only
`normal-formation.conf`. The 13 other phase files are unchanged
copies of the helios defaults, which means:

- During defensive phase (helios switches to `defense-formation.conf`
  when our team has lost the ball) the team reverts to the default
  4-back back line. The "5-back low block" the 3-2-5 was supposed
  to fall into never materializes; the wing-backs return to side-
  back positions.
- During offensive phase (`offense-formation.conf`) the team again
  reverts to default. The 3-up-top push that should happen with
  wing-backs joining the line never happens.
- Only during the "normal play" phase between possession
  transitions does the 3-2-5 shape appear.

A real Pep-style possession contrast needs all three phase files
rewritten in the same 3-2-5 / 5-3-2 spirit:
- `normal-formation.conf`  -> already done (this commit)
- `offense-formation.conf` -> 3-2-5 with WBs as wide forwards,
                              striker pinning the back line
- `defense-formation.conf` -> 5-3-2 low block (CBs + WBs + a
                              cover-shadow CDM pair, two narrow
                              forwards pressing the CBs)

Implementing the other two formation files is the obvious next
single-PR change. The harness will then re-run, the compare tool
will say `ALLOWED`, and the delta CI will either move off zero or
will continue to refuse the claim -- either is a research-grade
answer.

## Reproducing

```sh
export PATH="$PWD/externals/install/bin:$PATH"
export LD_LIBRARY_PATH="$PWD/externals/install/lib"
make real-smoke REAL_SMOKE_EXPERIMENT=experiments/helios_vs_3_2_5_smoke.yaml NUM_MATCHES=30
make compare \
  BASELINE=logs/experiments/helios_vs_helios_real_smoke/summary.json \
  VARIANT=logs/experiments/helios_vs_3_2_5_smoke/summary.json
```

Note that re-running the variant overwrites the lock-pinned external
versions only if `EXTERNALS.lock` is unchanged; that file is the
guarantee of reproducibility, so committed state of this repo
reproduces this exact result.
