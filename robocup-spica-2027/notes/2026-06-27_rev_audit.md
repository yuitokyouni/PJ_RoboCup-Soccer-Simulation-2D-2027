## 2. Phase 5 拡張モジュール (src/phase5/*.cpp 5本)

CMakeLists で phase5/*.cpp を glob して sample_player にリンク。

### 5a. `chance_signal.cpp` — 連続 chance signal bias

- Phase 5b 由来。`compute_chance_signal(wm)` を提供
- W_CONE 0.40 + W_MOM 0.30 + W_LANE 0.20 + W_PRESS 0.10 で構成
  - cone: 前方 30度コーン内に味方/敵相対密度
  - momentum: 直近 5 cycles のボール momentum
  - lane: 25m 前方への lane 通行可能性
  - press: 直近 20 cycles の press 履歴
- 値は 0..1 にクリップ。chain_action 内で per-candidate evaluator に bias 加算
  - cs > 0.5 + forward action → +30 * (cs-0.5)/0.5
  - cs < 0.3 + forward → -8 * (0.3-cs)/0.3
  - cs < 0.3 + hold/move → +6 * (0.3-cs)/0.3

### 5b. `bhv_smart_clearance.cpp` — 安全クリアランス + territory recovery 発火

- 守備時の clearance 行動を再定義
- ボール座標と相手プレス状況から「タッチ外蹴り出し」と「ダイレクト前方クリア」の選択
- クリア成功時に `TerritoryRecoveryState::activate()` 呼び、N cycles の前進バイアスを発動

### 5c. `territory_recovery_state.cpp` — クリア後の前進状態

- TRS singleton。`active(cycle)` で「最近クリアした」状態を返す
- active 中は defense_block の最後で全員 `shifted_x += forward_bias` する仕組み
- 守備後すぐの territory regain を狙う

### 5d. `counter_press_state.cpp` — counter-press 状態管理

- ボール奪取後 N cycles 間「counter press モード」に入る
- CPS singleton。`update(wm)` を bhv_basic_move::execute の冒頭で tick
- `is_recent_recovery_opp_half(cycle)` を chance_signal の press bonus が読む
- 既存マーク dash の倍率を上げる (Phase 5d wiring via bhv_block/mark)

### 5e. `defense_block.cpp` — 動的 4-3-3 / 3-2-5 / 5-2-3 切替

ファイル内に5周期改良の歴史 (Phase 5e v2, Phase 6, Phase 6.1revert, Phase 6.2, Phase 7 disable):

- `modulate_position(wm, unum, raw_target)`:
  - **attack phase** (味方ボール保持):
    - `is_wing_back(unum=3/4)` && `should_this_sb_push(unum, wm)` → 高い位置 (x = max(-5, ball.x-5), y = ±22) で wing-back 化
    - 片側 SB だけ押し上げ。stay-back SB は内側で 3-back の一翼
    - `is_build_up_drop_cdm(unum, wm)` (Phase 6 symmetric):
      - ball.y > 3 → unum 7 が落ちる, ball.y < 3 → unum 6 が落ちる
      - 落ちた CDM は反対側 (押し上げ SB の opposite flank) で 3-back 形成
    - `is_false_nine(unum=11)` の pocket run は **Phase 9 で DISABLED** (コメントアウト維持)
  - **defense phase**:
    - ball.x > 15 なら raw_target 返す (相手陣深いので追従しない)
    - 横シフト: `lateral_shift_amount(wm)` で ball-side に寄る (ball.y → raw_target.y 方向 + shift)
    - 縦圧縮: forwards (9/10/11) は midline (cap = max(0, ball.x+10)) まで下がる
    - SBs (3, 4) は 5-back 化: ball.x < -10 で min(shifted_x, -22), 以外 min(shifted_x, -14)
  - **territory recovery 適用**: 最後に TRS active なら全員 shifted_x に forward_bias 加算

### 5f. `intercept_discipline.cpp` (Phase 8)

- bhv_basic_move 内 Body_InterceptPlan 呼び出し前 gate
- CDM (unum 6/7) について「intercept 取りに行くと中央が空くか?」を判定
- 安全でない場合 InterceptPlan skip → fall through to formation positioning

## 5. ChainAction 拡張 (action_chain_graph.cpp Phase 5b)

`compute_chance_signal(wm)` で得た cs を per-candidate evaluator に bias 加算 (前述)。

加えて Phase 6 で 2 つの formation-aware bias:

- **wedge bias** (Pep / Nuno-Mendes パターン):
  - sender unum 3/4 (SB/WB), receiver unum 11 (CF), target.x > 0, |target.y| ∈ [6,16]
  - bonus +25

- **side-switch bias** (Pep verlagerung):
  - sender unum 2/3/4 (CB/SB), sender 自陣 (sp.x<0)
  - sp.y と tp.y の符号反転 (例: -5→+5)
  - |dy| >= 20, tp.x <= 25 (long ball over the top でなく switch)
  - bonus +15
